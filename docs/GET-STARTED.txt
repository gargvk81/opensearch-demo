ğŸ”¬ HOW OPENSEARCH WORKS

Indexing Process:
  1. App receives JSON document
  2. OpenSearch analyzes text:
     "High Performance Laptop" â†’ ["high", "perform", "laptop"]
  3. Creates inverted index:
     "high" â†’ [doc1, doc3]
     "perform" â†’ [doc1]
     "laptop" â†’ [doc1, doc5]
  4. Stores original document
  5. Document becomes searchable

Search Process:
  1. User queries "wireless mouse"
  2. Query analyzed: ["wireless", "mouse"]
  3. Inverted index searched
  4. Documents scored by relevance (BM25)
  5. Results sorted by score
  6. Top matches returned

Key Features:
  âœ“ Full-text search across multiple fields
  âœ“ Relevance ranking

ğŸ› ï¸ TROUBLESHOOTING

Problem: OpenSearch won't start
Solution:
  - Check Docker has enough memory (4GB+)
  - View logs: docker logs opensearch
  - Restart: docker-compose restart

Problem: Application can't connect to OpenSearch
Solution:
  - Verify OpenSearch is running: docker ps
  - Check credentials in application.yml match docker-compose.yml
  - Test connection: curl http://localhost:9200 -u admin:Admin@123 -k

Problem: Port already in use
Solution:
  - Check what's using the port: lsof -i :8080
  - Kill process or change port in application.yml
